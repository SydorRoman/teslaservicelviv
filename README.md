# Tesla Service Львів — лендінг

Статичний односторінковий сайт: HTML + CSS + Vanilla JS (ES6), GSAP + ScrollTrigger, Lenis.
Без збірки, без фреймворків, без npm-залежностей. Деплой — Cloudflare Pages.

```
index.html
css/    base.css · intro.css · sections.css
js/     main.js · intro.js · animations.js
assets/ images/ · icons/
functions/api/lead.js      Cloudflare Pages Function — приймання заявок
_headers  robots.txt  sitemap.xml
```

### Відгуки — окремо

Тексти в секції `#reviews` **вигадані** як приклад верстки, імена стоять як
`[Імʼя клієнта]`. Замініть на реальні. З тієї ж причини в розмітці Schema.org
**свідомо немає** `aggregateRating` і `review`: вигадані оцінки в структурованих
даних — це підробні свідчення й пряме порушення правил Google, за яке сайт
втрачає розширені сніпети. Додавайте їх лише разом із реальними відгуками.

### Логотип і назва — юридичний момент

Логотип у `assets/images/logo.svg` — **оригінальний** знак (блискавка в шестикутнику),
а не логотип Tesla, Inc. Це зроблено свідомо. Незалежний сервіс має право називати
марку, яку обслуговує (номінативне використання), але відтворення фірмового знака
виробника натякає на афіліацію. З тієї ж причини у футері стоїть дисклеймер про
незалежність — не видаляйте його.

## Деплой на Cloudflare Pages

1. **GitHub**

   ```bash
   git init && git add . && git commit -m "Tesla Service Lviv landing"
   gh repo create tesla-service-lviv --private --source=. --push
   ```

2. **Cloudflare Pages** → Create project → Connect to Git → оберіть репозиторій.

   | Налаштування | Значення |
   |---|---|
   | Framework preset | None |
   | Build command | *(порожньо)* |
   | Build output directory | `/` |

   Збірки немає — Pages просто роздає файли, а теку `functions/` підхоплює сам.

3. **Змінні оточення** (Settings → Environment variables) — тип **Secret**:

   | Змінна | Що це |
   |---|---|
   | `TELEGRAM_BOT_TOKEN` | токен від [@BotFather](https://t.me/BotFather) |
   | `TELEGRAM_CHAT_ID` | ID чату для заявок |

   Дізнатись `TELEGRAM_CHAT_ID`: напишіть боту будь-що, відкрийте
   `https://api.telegram.org/bot<TOKEN>/getUpdates`, візьміть `message.chat.id`.

   Без цих змінних `/api/lead` повертає 500, а форма показує людині телефон і
   Telegram як запасний канал. Це навмисно: мовчазний «успіх» = втрачені клієнти.

4. **Домен GoDaddy** → Cloudflare

   - Cloudflare → Add site → ваш домен → скопіюйте два NS-сервери.
   - GoDaddy → Domain Settings → Nameservers → Change → Enter my own → вставте NS.
   - Оновлення NS займає до 24 год (зазвичай ~1 год).
   - Pages → Custom domains → Set up a domain → додайте домен і `www`.
   - SSL/TLS → Overview → режим **Full (strict)**.

## Як влаштоване інтро

Логотип і атмосферна імла — **статичні**, видно в першому кадрі без жодної
анімації (звичайні CSS-правила в `css/intro.css`, JS їх не чіпає). Заголовок
(«Tesla Service» → «Львів» окремим рядком одразу за ним), підзаголовок і CTA
ховаються за замовчуванням і проявляються по `js/intro.js` через ScrollTrigger
з **коротким піном** (`PIN_VH_FRACTION` у `intro.js`, ≈1 екран).

**Чому пін, а не просто скрол-фейд.** Перша версія була взагалі без піна:
секція просто йшла в потоці документа, і елементи фейдились по мірі скролу.
На мобільному це виявилось нежиттєздатним — один свайп зі своєю інерцією
проскакував і reveal, і саму секцію одним рухом, тож CTA-кнопка ніколи не
зупинялась у стабільному положенні на екрані, і по ній неможливо було влучити
пальцем. Пін гарантує: поки не проскролено всю відстань `PIN_VH_FRACTION`,
сторінка не йде далі — reveal завершується десь на 60% цієї відстані
(порожній твін-«хвіст» у кінці таймлайну), а решту ~40% CTA просто стоїть
нерухомо й чекає на тап.

**Заміна тексту.** Обидва рядки заголовка — прості `<span>` у розмітці
(`index.html`, `#intro-title`), кожен зі своєю маскою (`.intro__title-row`).
Щоб додати третій рядок або змінити порядок — правте розмітку й `.intro__line`
підхопить його автоматично (JS звертається до `.intro__line` через
`querySelectorAll`, без хардкоду кількості).

**Неочевидне місце — не зламайте.** `z-index: 2` на `.intro` в `intro.css`.
Під час піна сцена стає `position: fixed`, а всі `.section` мають
`position: relative`. Серед позиціонованих елементів із `z-index: auto`
виграє останній у DOM — без цього правила секція «Про нас» замальовує
собою запінену сцену, і екран під час піна просто чорний.

## Секція «Наші фото»

Горизонтальний скролер реальних фото сервісу (`#photos` в `index.html`),
окремо від сітки-заглушки в «Наша майстерня». Нативний `overflow-x` +
`scroll-snap-type: x proximity` — без бібліотек каруселі. `js/main.js`
(`initPhotostrip`) додає лише те, чого CSS сам не вміє: перетворення
вертикального колеса миші на горизонтальний скрол (з відпусканням жесту
сторінці на краях стрічки) і кнопки-стрілки.

**Чому `proximity`, а не `mandatory`.** `mandatory` форсує прив'язку на
кожне присвоєння `scrollLeft`, навіть програмне на кілька пікселів — через
це погортання колесом миші «стрибало» б назад до найближчої картки замість
плавного руху. `proximity` прив'язує лише коли скрол уже зупинився поруч
із межею картки.

**Фото:** оригінали — HEIC з телефону (`~/pic`), сконвертовані через
Pillow: EXIF/GPS видалено (приватність), стиснуто в JPEG q76 + WebP q72,
макс. сторона 1280px. Щоб додати ще фото — покладіть `.jpg`/`.webp` в
`assets/images/photos/` і додайте `<li class="photostrip__item">` за тим
самим шаблоном у розмітці; JS/CSS не потребують змін.

## Продуктивність

- Нуль вебшрифтів — системний стек (SF Pro на Apple, Segoe на Windows).
- Іконки — інлайн-SVG-спрайт, без окремого запиту.
- Карта Google вантажиться лише по кліку (iframe коштує ~10–15 балів Lighthouse).
- Фото — `loading="lazy"` + `width`/`height` (інакше поїде CLS).
- Анімуються лише `opacity` і `transform`; свічення — заздалегідь розмиті
  градієнти, яким міняється прозорість, а не `filter: blur()` на таймлайні.

Що ще можна зробити на проді:

- Звантажити GSAP і Lenis у `js/vendor/` замість CDN (мінус 2 DNS-запити).
- Конвертувати фото в AVIF/WebP і додати `srcset`.

## Доступність

- `prefers-reduced-motion` вимикає інтро й Lenis, показуючи фінальний кадр одразу.
- Без JS (або якщо CDN недоступний) клас `no-js` повертає весь контент у видимий стан.
- Кнопка «Пропустити інтро», skip-link, видимий фокус із клавіатури.
